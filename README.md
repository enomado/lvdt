# LVDT Conditioner 🧲📐

Прошивка для сигнального кондиционера LVDT на STM32G474CEU6, целиком на Rust + RTIC.
Цель — превратить аналоговый дифференциальный датчик в честный цифровой поток
позиции, без жирной DSP‑матчасти и без C.

## Идея ✨

LVDT — это трансформатор с подвижным сердечником. Подаём синус на первичку,
со вторичек снимаем две амплитудно‑модулированные огибающие, считаем разность
и делим на сумму — получаем положение. Всё работает на одной частоте возбуждения,
поэтому самая чистая демодуляция — синхронная (IQ): умножаем сигнал на cos/sin
той же фазы, что и возбуждение, и усредняем.

```text
TIM6 TRGO @ ~160 kHz  ⏱️
  → DAC1_CH1 (PA4) — синус возбуждения, 64 точки на период  🌊
  → ADC1 + ADC2 dual regular simultaneous                   🎚️🎚️
  → DMA, циркулярные блоки по 64 пары отсчётов              🔁
  → IQ‑демодуляция блока (1 период возбуждения)             ➗
  → когерентное усреднение I/Q по N блокам, |Z|, ΔA−B       Σ
  → USB CDC, текстовый поток позиции                        🔌
```

## Принципы 🧠

- **Rust + RTIC, no_std.** 🦀 Никакого heap‑аллокатора, никаких глобальных
  `static mut`. Ресурсы и приоритеты прерываний описаны декларативно, гонок
  данных просто нет — компилятор не даст.
- **Аппаратные таймеры тащат всё.** ⛓️ Период выборки задаёт TIM6 → TRGO,
  он же триггерит и DAC, и ADC. Фаза между возбуждением и выборками
  фиксирована железом, а не software‑таймингом.
- **DMA и ноль копий.** 🚚 ADC пишет packed‑пары `(ADC1<<16)|ADC2` сразу
  в кольцевой буфер; CPU видит только готовые блоки и считает по ним IQ.
- **Фиксированный LUT синуса.** 📐 64 точки на период — ровно столько,
  чтобы IQ‑опоры (`cos`, `sin` под ту же сетку) считались как сдвиг индекса,
  без `sinf()` в реальном времени.
- **Числа целые.** 🔢 Демодуляция в `i32`, нормировка по сумме каналов
  даёт результат, инвариантный к амплитуде возбуждения и дрейфу питания.
- **Когерентное усреднение I/Q.** Σ Складываем `I` и `Q` последних `N`
  блоков до вычисления `|Z|` — это математически эквивалентно демодуляции
  окном `N·64` сэмплов с тем же LUT, повторённым `N` раз. Шум усредняется
  как √N (узкая полоса детектора ≈ `EXCITATION_HZ / N`), систематика и
  калибровка не сдвигаются — то есть растёт *повторяемость* (precision),
  а не привязка к нулю (accuracy для этого нужна отдельная калибровка
  механического центра и температурный коэффициент). Платим групповой
  задержкой `N · 64 / SAMPLE_HZ` секунд и сужением полосы отслеживания
  движения сердечника. Сейчас `N=64` (≈25.6 мс окно, шум −18 дБ к
  одиночному блоку); по мере роста требований к разрешению и при
  стационарных режимах планируем поднимать до `128…512`. Накопитель
  живёт в `iq::Accumulator`, отдельно от `demodulate_block`, чтобы
  параметр `N` менялся одной константой `SMOOTHING_BLOCKS` в `main.rs`.
- **Хост‑тестируемая математика.** 🧪 Распаковка ADC и IQ живут в `iq.rs`
  и собираются под обычный target, чтобы гонять `cargo test` без железа.
- **Слои разнесены по файлам.** 🧩 `clocks`, `excitation`, `sampling`, `iq`,
  `usb_cdc` — каждый отвечает за один кусок цепи и общается через типы,
  а не через глобалы.

## USER‑кнопка 🔘

Одна кнопка K1 (PC13, **active HIGH** — на этой плате нестандартно, см. [PROGRESS.md](PROGRESS.md)) распознаёт четыре паттерна нажатий «как у фонариков» и переключает режим OLED‑экрана. FSM в [src/button.rs](src/button.rs) опрашивается каждые 8 мс из async RTIC‑таска, дебаунс 15 мс, порог short/long — 500 мс, окно между кликами серии — 350 мс.

| Жест                          | Паттерн | Экран        | Что показывает                                            |
|-------------------------------|---------|--------------|------------------------------------------------------------|
| короткий клик                  | `S`     | **Status**   | `A.x4 100.0%` / `B.x4 95.0%` — магнитуда + Q + gain        |
| два коротких подряд            | `SS`    | **Phase**    | `A.x4 +12d` / `B.x4 -8d` — фаза в градусах к DAC           |
| удержание ≥0.5 с               | `L`     | **Position** | `POS +12.34%` крупно + `A.x4 B.x16` снизу, gain‑normalized |
| длинный + два коротких         | `LSS`   | **Raw**      | `AS000 ~99.9%` / `BS001 ~80.5%` — sat_count + sine purity  |

Нераспознанные комбинации (`SSS`, `LL`, `LS`, длиннее 4 кликов) тихо отбрасываются после `INTERCLICK_GAP_MS` — это «escape», если передумал. Стартовый режим после reset — `Status`. На каждом распознанном паттерне в RTT летит `defmt::info!("button: SS → screen=phase")`.

## Сборка и запуск 🛠️

Один раз поставить таргет:

```sh
rustup target add thumbv7em-none-eabihf
```

Проверить прошивку:

```sh
cargo check
```

Хост‑тесты для математики:

```sh
cargo test --target x86_64-unknown-linux-gnu
```

Прошить и запустить через probe‑rs:

```sh
cargo run --release
```
